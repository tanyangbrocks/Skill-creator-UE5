# 採掘建築系統設計

*建立：2026-06-09*

參考：Minecraft、Terraria。涵蓋使用者需求 + 補充項目。

---

## 一、設計原則

- **架構優先**：不立即實作的功能也要預留資料欄位，避免日後大改介面。
- **物品欄統一**：玩家、敵人、容器方塊（箱子）共用同一個 `Inventory` 類別。
- **與法陣系統整合**：爆炸、Raycast 採掘是否掉落物品，由同一套掉落邏輯處理。

---

## 二、使用者原始需求

1. 所有非生物元素的「可挖掘 / 不可挖掘 / 硬度」屬性；預留抗爆度、抗魔度擴充空間。
2. 挖掘後變成「掉落物」（懸浮地面附近），供附近有物品欄的實體拾取。
3. 玩家擁有物品欄，可拾取並存放掉落物。
4. 部分物品可讓玩家放置、使用。
5. 整體仿照 Minecraft / Terraria。

---

## 三、補充項目（參考 Minecraft / Terraria）

以下為兩款遊戲中使用者尚未明確提到、但架構上應預先考慮的部分：

### 3.1 採掘機制補充

| 項目 | 說明 | 實作時機 |
|------|------|---------|
| **採掘速度** | 硬度決定需要幾幀才能挖完；工具有速度加成係數 | Phase D |
| **工具分類** | 鎬（石/礦）、斧（木）、鏟（土/沙）；對對應材質有效率加成 | Phase F（工具系統）|
| **工具等級（鎬力）** | 某些硬材質需要最低工具等級才能開採（Terraria 的鎬力 %） | Phase F |
| **採掘範圍** | 玩家只能對一定距離內的格子採掘/放置（約 4–5 格） | Phase D |
| **採掘進度回饋** | 被採掘的格子顯示進度（顏色漸暗 / 裂痕 overlay / 數字）| Phase D |
| **按住連續採掘** | 按住左鍵持續對同一格施工，鬆開重置進度 | Phase D |
| **自動目標** | 游標附近最近的可採掘格優先命中（Terraria smart cursor）| 延後 |

### 3.2 物品/掉落補充

| 項目 | 說明 | 實作時機 |
|------|------|---------|
| **物品堆疊** | 同種物品堆疊至上限（預設 99）| Phase B |
| **掉落物存活時間** | 不撿則 60 秒後消失，避免世界堆積 | Phase C |
| **掉落物重力** | 掉落物從方塊位置落到地面，漂浮小幅上下振盪 | Phase C |
| **自動拾取半徑** | 玩家 2 格以內自動吸取掉落物 | Phase C |
| **稀有/機率掉落** | 部分材質有額外稀有掉落（如岩石偶爾掉燧石）| 延後 |
| **挖掘≠材質** | 石頭挖後掉「圓石」、樹木掉「木材」（物品 ≠ 材質 1:1）| Phase B |

### 3.3 物品欄補充

| 項目 | 說明 | 實作時機 |
|------|------|---------|
| **熱鍵欄** | 物品欄的前 N 格（5–10）顯示在畫面底部，玩家直接切換 | Phase E（HUD）|
| **槽位上限** | 固定格數（建議初期 30 格：5 熱鍵 + 25 主欄）| Phase B |
| **工具耐久** | 工具使用次數耗損，歸零損壞 | 延後 |

### 3.4 放置機制補充

| 項目 | 說明 | 實作時機 |
|------|------|---------|
| **放置驗證** | 目標格必須為 Air、不能與玩家/實體重疊 | Phase E |
| **放置範圍** | 同採掘範圍（約 4–5 格）| Phase E |
| **容器方塊（箱子）** | 放置後本身擁有物品欄，玩家互動可打開 | 延後 |
| **背景牆層** | 前景（實體碰撞）+ 背景（純裝飾，不擋人）雙層 | 延後 |

## 四、材質屬性擴充（MaterialData）

### 4.1 需新增欄位

```csharp
// 在 MaterialData record 新增：
bool   IsMineable          // false = 不可採掘（如岩床 Bedrock）
int    Hardness            // 0 = 瞬間，越大越難挖；單位：幀數（基礎速度）
int    RequiredToolTier    // 0 = 徒手，1 = 基礎工具，2 = 進階工具，…
float  BlastResistance     // 爆炸傷害係數，預設 1.0f（保留）
float  MagicResistance     // 魔法傷害係數，預設 1.0f（保留）
ItemDrop[] DefaultDrops    // 採掘後掉落表
```

### 4.2 掉落表結構

```csharp
struct ItemDrop
{
    public ItemId ItemId;   // 掉落物種類
    public int    MinCount; // 最少數量
    public int    MaxCount; // 最多數量
    public float  Chance;   // 掉落機率 0–1（1 = 必掉）
}
```

### 4.3 各材質初始值（草稿）

| 材質 | IsMineable | Hardness | RequiredToolTier | 掉落 |
|------|-----------|---------|-----------------|------|
| Air | false | — | — | — |
| Dirt | true | 15 | 0（徒手可） | 1 泥土 |
| Stone | true | 40 | 1（基礎工具）| 1 圓石 |
| Wood | true | 20 | 0 | 1 木材 |
| Sand | true | 10 | 0 | 1 沙 |
| Water | false | — | — | — |
| Lava | false | — | — | — |
| Ash | true | 5 | 0 | 1 灰 |
| Bedrock（底層石）| false | — | — | — |

## 五、物品系統

### 5.1 ItemId 枚舉

```csharp
public enum ItemId
{
    None = 0,
    // ── 方塊物品（可放置）────────────────────────────────
    BlockDirt,
    BlockStone,       // 圓石（石頭挖出的物品）
    BlockWood,
    BlockSand,
    // ── 工具（採掘用）────────────────────────────────────
    ToolBasicPick,    // 基礎鎬
    ToolBasicAxe,     // 基礎斧
    // ── 材料（合成用，暫存）─────────────────────────────
    MaterialAsh,
    // ── 預留 ─────────────────────────────────────────────
    // 未來：藥水、武器、飾品、稀有礦石…
}
```

### 5.2 ItemStack

```csharp
public struct ItemStack
{
    public ItemId ItemId;
    public int    Count;
    public bool   IsEmpty => ItemId == ItemId.None || Count <= 0;
    public static readonly ItemStack Empty = new(ItemId.None, 0);
}
```

### 5.3 物品元資料（ItemData）

```csharp
public record ItemData(
    string DisplayName,
    bool   IsPlaceable,         // 可放置為方塊
    MaterialType? PlaceAs,      // 放置時變成哪種材質
    bool   IsTool,              // 工具
    int    ToolTier,            // 工具等級（對應 RequiredToolTier）
    float  MiningSpeedMult,     // 採掘速度倍率（1.0 = 基準）
    int    MaxStack             // 堆疊上限（工具類 = 1）
)
```

## 七、掉落物系統（DroppedItem）

### 7.1 資料結構

```csharp
public class DroppedItem
{
    public GridPos   Position { get; set; }
    public ItemStack Stack    { get; set; }
    public float     LifeTime { get; set; } = 60f;  // 秒，歸零後消失
    public bool      IsAlive  => LifeTime > 0f;

    // 物理
    private float _floatOffset = 0f;  // 0–1 視覺振盪相位
    private float _fallTimer   = 0f;
    private const float FallInterval = 0.15f;
}
```

### 7.2 DroppedItemManager

```csharp
public class DroppedItemManager
{
    private readonly List<DroppedItem> _items = new();

    // TileWorld.OnTileDestroyed 的回應：生成掉落物
    public void Spawn(GridPos pos, MaterialType mat) { … }

    // 每幀更新：重力、存活時間、自動拾取
    public void Update(TileWorld world, PlayerController player, float delta) { … }
}
```

### 7.3 整合點

- `TileWorld.OnTileDestroyed` 事件 → `DroppedItemManager.Spawn(pos, mat)`
- 爆炸（`TileWorld.Explode`）目前直接清除格子，不觸發 `OnTileDestroyed`。
  → **設計選擇**：爆炸是否掉落物品？建議爆炸不掉落（符合 Minecraft 邏輯），
    但可加 `Explode(dropItems: bool)` 參數保留彈性。

## 八、採掘機制

### 8.1 流程

```
玩家按住左鍵
  → 計算目標格（滑鼠世界座標）
  → 檢查距離 ≤ MiningRange
  → 檢查 IsMineable && RequiredToolTier ≤ 玩家工具等級
  → 每幀累加採掘進度
  → 進度 ≥ Hardness → DestroyTile → 觸發掉落
  → 鬆開或移開目標格 → 重置進度
```

### 8.2 採掘進度追蹤

```csharp
// PlayerController 內
private GridPos? _miningTarget  = null;
private float    _miningProgress = 0f;

// 採掘速度（幀數 = Hardness / (工具速度 × 工具對材質效率)）
// 暫時簡化：每幀固定 +1，Hardness 即需要的幀數
```

### 8.3 視覺回饋

在 TileWorldRenderer 渲染時，對 `_miningTarget` 格加上進度 overlay：

| 進度 % | 表現 |
|--------|------|
| 0–30% | 格子略暗 |
| 30–60% | 明顯暗色 |
| 60–100% | 非常暗（即將破壞）|

## 九、放置機制

### 9.1 流程

```
玩家按右鍵
  → 檢查熱鍵欄 ActiveItem.IsPlaceable
  → 計算目標格（滑鼠世界座標）
  → 檢查距離 ≤ MiningRange
  → 檢查目標格 == Air
  → 檢查目標格不與玩家/實體重疊
  → world.Set(target, item.PlaceAs)
  → inventory.Consume(activeSlot)
```

### 9.2 與現有滑鼠繪製的關係

目前 TileWorldRenderer 已有左鍵繪製 / 右鍵清除的「創作者工具」。
採掘/放置是「遊玩模式」邏輯，兩者可以用模式切換（或改為編輯器模式限定）區分。
**建議**：編輯器模式下保留現有畫筆工具；遊玩模式下由 PlayerController 控制採掘/放置，
TileWorldRenderer 的畫筆工具在遊玩模式禁用。

| Phase | 內容 | 關鍵檔案 |
|-------|------|---------|
| **A** | MaterialData 擴充欄位（IsMineable, Hardness, RequiredToolTier, BlastResistance, MagicResistance, DefaultDrops）| `Materials/MaterialData.cs` / `MaterialRegistry.cs` |

## 十二、暫不實作（保留架構）

| 項目 | 原因 |
|------|------|
| 工具耐久 | 需要平衡，先驗證核心採掘手感 |
| 背景牆層 | 需要 TileWorld 雙層架構，影響範圍較大 |
| 容器方塊（箱子） | 需要 TileEntity 概念（方塊本身有物品欄），留待 NPC 系統後一起考量 |
| 稀有/機率掉落 | ItemDrop.Chance 欄位已預留，後續填入數值即可 |
| 自動目標 (smart cursor) | 先做基本的滑鼠格選取，之後加 |
| 合成系統 | 使用者仍在構思設計形式 |
| 貨幣系統 | 暫無設計 |
