# 地圖生成改版計畫：Heightmap + Cellular Automata 混合

> 目標：取代現有 BSP 地牢布局，改為「地表（Heightmap）+ 地下洞穴（CA）」的分層混合地圖。

---

## 可行性評估

| 面向 | 結論 |
|------|------|
| 技術可行性 | ✅ 高。兩個演算法都對同一個 `TileWorld` 格子陣列操作，順序執行即可 |
| 介面契約相容 | ✅ `SpawnData`（PlayerSpawn + EnemySpawns）結構不變，TileWorld 完全不動 |
| 需改動的檔案 | 只有 `Scripts/World/MapGenerator.cs`（完整替換）|
| `Room struct` | ✅ 可直接刪除，Heightmap + CA 不需要「房間」概念 |
| 生成速度 | ✅ CA 5輪迭代 × W×H 格子 ≈ 1ms；Heightmap O(W) 雜訊取樣，比 BSP 還快 |

---

## 地圖分區結構

```
Y = 0      ┌──────────────────────────────────────────────┐
           │                  天空（Air）                   │  ← 永遠空氣
           │                                              │
Y = sMin   ├~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~┤  ← 最高地表點
           │     地表區（Heightmap 控制輪廓）              │
           │   ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░   │  ← Dirt/Stone
Y = sMax   ├──────────────────────────────────────────────┤  ← 最低地表點
           │                                              │
           │       地下洞穴區（CA 生成）                   │
           │   ○○○ ○ ○  ○○○○  ○   ○○   ○○○  ○  ○        │  ← 有機洞穴
           │  ○○○○○○  ○○○○○○○○○○  ○○○○○○○○○○○○○○○       │
           │                                              │
Y = H-8    ├──────────────────────────────────────────────┤
           │              底部岩床（Stone）                 │
Y = H      └──────────────────────────────────────────────┘
```

- **sMin**（最高地表）≈ `H × 0.28`
- **sMax**（最低地表）≈ `H × 0.42`
- **CA 活動區**：`sMax` 以下到 `H-8`
- **過渡帶**（`sMin ~ sMax`）：Heightmap 優先，CA 補洞

## 實作步驟

### Step 1 — 替換 MapGenerator.cs 結構

刪除 `Room struct` 和所有 BSP 方法（`BspSplit`、`CarveRoom`、`CarveCorridor`）。  
保留 `SpawnData`、`Generate` 方法簽名、`AddDecor`（改寫）。

主流程改為：

```csharp
public static SpawnData Generate(TileWorld world, int seed = 12345)
{
    var rng = new Random(seed);
    int W = world.Width, H = world.Height;

    // 1. 全部填石
    FillAll(world, W, H, MaterialType.Stone);

    // 2. Heightmap → 塑形地表
    int[] heights = GenerateHeightmap(W, H, rng);
    ApplyHeightmap(world, heights, W, H);

    // 3. CA → 挖掘地下洞穴
    bool[,] caves = GenerateCaCaves(W, H, heights, rng);
    ApplyCaves(world, caves, heights, W, H);

    // 4. 修補：確保洞穴與地表有至少一條通路
    GridPos surfaceEntry = EnsureConnectivity(world, heights, W, H, rng);

    // 5. 底部岩床
    SealBedrock(world, W, H);

    // 6. 裝飾
    AddDecor(world, heights, W, H, rng);

    // 7. 生成點
    return BuildSpawns(world, heights, surfaceEntry, W, H, rng);
}
```

### Step 2 — Heightmap 生成（`GenerateHeightmap`）

```csharp
private static int[] GenerateHeightmap(int W, int H, Random rng)
{
    // 基礎高度：H 的 32%
    float baseY = H * 0.32f;

    // 3 層疊加正弦 + 隨機漂移，模擬自然地形起伏
    float[] raw = new float[W];
    float phase1 = rng.NextSingle() * MathF.Tau;
    float phase2 = rng.NextSingle() * MathF.Tau;
    float phase3 = rng.NextSingle() * MathF.Tau;

    for (int x = 0; x < W; x++)
    {
        float fx = (float)x / W;
        raw[x] = baseY
            + MathF.Sin(fx * 2 * MathF.PI + phase1) * (H * 0.05f)   // 大波
            + MathF.Sin(fx * 7 * MathF.PI + phase2) * (H * 0.025f)  // 中波
            + MathF.Sin(fx * 17 * MathF.PI + phase3) * (H * 0.012f) // 小細節
            + (rng.NextSingle() - 0.5f) * (H * 0.012f);             // 雜訊
    }

    // 平滑 3 次（消除尖刺）
    for (int pass = 0; pass < 3; pass++)
        for (int x = 1; x < W - 1; x++)
            raw[x] = (raw[x - 1] + raw[x] + raw[x + 1]) / 3f;

    // 轉換為整數格子高度
    int[] heights = new int[W];
    for (int x = 0; x < W; x++)
        heights[x] = Math.Clamp((int)raw[x], (int)(H * 0.20f), (int)(H * 0.45f));

    return heights;
}
```

`ApplyHeightmap`：高於 `heights[x]` 的行設為 Air；`heights[x]` 那一格設為 Dirt；往下 2 格也設 Dirt（土壤層），再往下保留 Stone。

### Step 3 — CA 洞穴生成（`GenerateCaCaves`）

```csharp
private static bool[,] GenerateCaCaves(int W, int H, int[] heights, Random rng)
{
    // 只在地表以下的區域進行 CA
    int caveTop = heights.Max() + 3;   // 稍微低於最深地表

    var cells = new bool[W, H];   // true = 空氣（洞穴）

    // 初始化：45% 機率為空氣
    for (int y = caveTop; y < H - 8; y++)
    for (int x = 0; x < W; x++)
        cells[x, y] = rng.NextSingle() < 0.45f;

    // 4 輪 CA smoothing（標準 B5678/S45678 洞穴規則）
    for (int step = 0; step < 4; step++)
        cells = SmoothCa(cells, W, H, caveTop);

    // 最後 1 輪更激進平滑（填補細碎孤島）
    cells = SmoothCa(cells, W, H, caveTop, aggressiveThreshold: 4);

    return cells;
}

private static bool[,] SmoothCa(bool[,] cells, int W, int H, int caveTop,
    int aggressiveThreshold = 5)
{
    var next = (bool[,])cells.Clone();
    for (int y = caveTop; y < H - 8; y++)
    for (int x = 0; x < W; x++)
    {
        int stoneNeighbors = 0;
        for (int dy = -1; dy <= 1; dy++)
        for (int dx = -1; dx <= 1; dx++)
        {
            int nx = x + dx, ny = y + dy;
            if (nx < 0 || nx >= W || ny < 0 || ny >= H || ny < caveTop)
                stoneNeighbors++;   // 越界算 Stone
            else if (!cells[nx, ny])
                stoneNeighbors++;
        }
        // ≥ threshold 個石頭鄰居 → 變石頭（填死）；< threshold → 變空氣（開洞）
        next[x, y] = stoneNeighbors < aggressiveThreshold;
    }
    return next;
}
```

`ApplyCaves`：把 `cells[x,y] == true`（洞穴空氣）對應的 tile 設為 Air，但不覆蓋 `heights[x]` 以上（地表輪廓保持）。

### Step 4 — 連通性保證（`EnsureConnectivity`）

```csharp
private static GridPos EnsureConnectivity(TileWorld world, int[] heights, int W, int H, Random rng)
{
    // 在地表中央找一個玩家落腳點
    int midX    = W / 2;
    int spawnY  = heights[midX] - 1;  // 地表正上方第一格 Air
    var start   = new GridPos(midX, spawnY);

    // Flood-fill 從地表往下，記錄可到達的格子
    var visited = FloodFill(world, start, W, H);

    // 找出 CA 洞穴中沒有被 visited 的最大連通區
    // 若存在：從 visited 區最近的石頭往下垂直打通一條 2 格寬的豎井
    int surfaceDeep = heights.Max() + 8;
    for (int x = midX - W / 4; x < midX + W / 4; x++)
    {
        for (int y = surfaceDeep; y < H - 10; y++)
        {
            if (world.Get(x, y).Type == MaterialType.Air && !visited.Contains(new GridPos(x, y)))
            {
                // 找到孤立洞穴 → 從地表打通豎井
                for (int sy = heights[x]; sy <= y; sy++)
                    for (int dx = -1; dx <= 1; dx++)
                        world.Set(x + dx, sy, MaterialType.Air);
                break;
            }
        }
    }

    return start;
}
```

> 簡化版：只打通一條豎井確保最小連通性。後續可改為完整多區域 flood-fill 修補。

### Step 5 — 裝飾改寫（`AddDecor`）

不再依賴 Room 清單，改為掃描地圖：

- **地表草皮**：掃描每列，找到最上層 Dirt，在其正上方 Air 格放 Grass 裝飾
- **石鐘乳**：掃描洞穴天花板（上方是 Stone，本身是 Air），隨機放置 1-3 格向下的石棍（MaterialType.Stone）
- **水坑**：洞穴底部（本身 Air，下方 Stone）隨機填水
- **木台**：洞穴內高度 ≥ 8 的開放空間隨機加木頭平台（邏輯同 BSP 版，改為位置掃描）

---

### Step 6 — 生成點改寫（`BuildSpawns`）

```
玩家生成點：
  → 地圖中央附近，heights[midX] - 1（地表上第一格 Air）

敵人生成點：
  → 在 CA 洞穴區（y > heights.Max() + 4）Flood-fill 找到最大連通區域
  → 在最大區域內選 5 個分散的地面格（本身 Air，下方 Stone）
  → 依位置由左到右分配型別（與 BSP 版相同邏輯）
```

## 分階段工作順序（建議）

1. **Phase A**（核心結構）：實作 `GenerateHeightmap` + `ApplyHeightmap` → 地表可見  
2. **Phase B**（地下洞穴）：實作 `GenerateCaCaves` + `ApplyCaves` → 洞穴出現  
3. **Phase C**（連通修補）：實作 `EnsureConnectivity` → 確保可到達洞穴  
4. **Phase D**（生成點）：改寫 `BuildSpawns` → 玩家/敵人正確生成  
5. **Phase E**（裝飾）：改寫 `AddDecor` → 視覺潤飾  

每個 Phase 完成後可以直接在 Godot 裡跑地圖確認效果，失敗不影響其他系統。

## 備注

- Heightmap 的「波形」參數（振幅、頻率）可以抽成 public const 方便調整
- CA 的初始密度（45%）和門檻（5）是標準值，可調整：密度越高 → 石牆越多，門檻越低 → 洞穴越大
- 如果不希望看起來是全地下洞穴風格，可以把 caveTop 調高（讓 CA 只在更深的地方運作）
- `FastNoiseLite`（Godot 內建）可取代正弦疊加，但需要注意 Godot 物件不能在靜態方法中用 `new FastNoiseLite()` 初始化（需要在場景樹節點上使用），所以暫時用純 C# 正弦疊加
